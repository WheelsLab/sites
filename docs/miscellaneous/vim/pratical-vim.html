vim 实用技巧

使用出厂设置启动 vim

$ vim -u NONE -N

essencial.vim
set nocompatible
filetype plugin on

使用指定配置文件启动 vim

$ vim -u code/essential.vim

Vim 的思考方式​

技巧 1

Meet the Dot Command

. 用于重复上一次修改，是 Vim 中最强大、最通用的编辑命令之一。

这里的“修改”（change）指的是会改变缓冲区内容的操作，比如：

  • x 删除一个字符
  • dw 删除一个单词
  • cwfoo<Esc> 把一个词改成 foo
  • >> 缩进一行
  • J 合并行

执行一次修改后，再按 .，Vim 会原样重放这次修改。

. 把“编辑”变成“可复用的操作”。我只改一次，但要让这一次“可重复”

. 命令是一个小型宏

技巧 2

Don’t Repeat Yourself

对于常见操作，比如在一行末尾添加分号，Vim 提供了专门的复合命令，把移动和修改合并为一步操作（例如 A 等同于 $a）。

减少不必要的移动

a：在当前光标后追加文本

A：在当前行末尾追加文本

一次键击用于移动，一次键击用于执行操作。

一次按键，完成原本需要两步的操作

A 并不特殊——Vim 里有一整类这样的“复合命令”。

这些单条、语义完整、会进入 Insert 的复合命令把一次修改压缩成一次可重复的 change。正因为如此，它们与 . 命令天然契合。构成 Vim 高效编辑的核心机制。

Longhand复合命令
c$C
cls
^CS
^iI
$aA
A<CR>o
kOO
技巧 3

Take One Step Back, Then Three Forward

使用 Vim 的地道技巧，我们可以在单个字符前后添加空格。起初看起来有点古怪，但优势是可重复，极大提高编辑效率。

比如我们要在 + 号前后添加空格

var foo = "method("+argument1+","+argument2+")"
  • 操作流程：
    1. 查找字符：f+ 移动到下一个 +
    2. 修改动作：s+ <Esc>
    3. 重复操作：;（重复 f+），.（重复上一次修改）

让移动变得可重复

f{char} 命令，移动到指定字符下一次出现的位置。

; 可以重复上一次字符查找

全部放在一起

; 将我们带向下一个目标，. 命令重复上次的 change。

技巧 4

Act, Repeat, Reverse

面对重复性任务，只要让“移动（motion）”和“修改（change）”都可重复，就能获得最优的编辑策略。Vim 天生擅长这一点，它记住操作，并把常用命令放在触手可及的位置，轻松重放。

在当前行查找下一个字符f{char} / t{char};,
在当前行查找上一个字符F{char} / T{char};,
在文档中查找下一个匹配项/pattern<CR>nN
在文档中查找上一个匹配项?pattern<CR>nN
执行替换:s/target/replacement&u

技巧 5

Find and Replace by Hand

Vim 提供 :substitute 命令用于查找替换，但还有一种手动方式：先修改第一个匹配项，然后逐个跳转并替换。. 命令可以自动重复修改，让操作更轻松，同时配合跳转命令提高效率。

...We're waiting for content before the site can go live...
...If you are content with this, let's go ahead with it...
...We'll launch as soon as we have the content...

  • *：搜索当前光标所在单词

  • 手动修改第一个匹配项

  • 用 n 跳转下一个匹配项

  • . 重复修改

技巧 6

Meet the Dot Formula

前面三个技巧中，每个问题不同，但都可用 . 命令解决。在这里总结出一个共通规律——我称之为“点范式”。

核心模式：一次键击移动，一次键击执行操作

  • 技巧 2：Don’t Repeat Yourself 在每一行末尾添加 ;，最终用 j. 自动化操作

  • 技巧 3：Take One Step Back, Then Three Forward 在 + 号前后添加空格，最终用 ;. 自动化操作

  • 技巧 5：Find and Replace by Hand 用 copy 替换部分 content，最终用 n. 自动化操作

这种模式在 Vim 中会反复出现：One Keystroke to Move, One Keystroke to Execute。

note

使用 vim 时我们在对文本进行编程，而不是单纯地输入字符。编程讲究复用。

Vim 高效的原因在于避免重复劳动（移动和修改都可复用），它通过以下机制实现这一点：

  1. . 命令——重复上一次修改，把一次编辑变成可复用的操作。
  2. 复合命令（如 A、C、I 等）——把移动和修改合并为一步操作，减少多余的键击。
  3. 可重复的移动（f{char}、; 等）——让光标移动也能复用，而不仅仅是修改动作。
  4. 点范式（Dot Formula）——遵循“一次键击移动，一次键击执行”的模式，使复杂操作可以快速循环应用。

Mode（模态编辑器）​

Vim 被称为 模态编辑器（modal editor），因为每次按键的效果都取决于当前模式。它将操作和输入分离，主要有普通模式（用于移动光标、删除、复制等命令）、插入模式（用于输入文本）、可视模式（用于选择文本）以及命令行模式（用于保存、搜索、替换等操作）。因此，清楚自己处于哪种模式、知道如何在模式之间切换，以及理解每个模式的功能，对于高效使用 Vim 非常重要。
