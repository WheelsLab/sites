Vim 入门指南

vi 是Linux下的文本编辑器，同时VI是罗马数字6，平时我们会说666，表示很厉害，所以 vi 就是很牛/🐮/6的编辑器

vi 中命令是操纵文本的工具，移动光标？请用命令 hjkl。要插入文本？请用命令 i。搜索文本？请用命令 /。要跳转到文本最后？请用命令 G。要退出编辑器？请用命令 wq。

我在哪输入命令呢？ :wq

这不是颜文字，是叫你先输入: 然后输入wq 最后按 Enter，这个是保存并退出的命令，就是一般编辑器的保存并退出（save and quit）功能（不知道多少人会卡在关闭编辑器上，笑.jpg）。

很奇特吧？vi 具有和其他所有文本编辑器都不一样的操作模式。这就叫模态编辑器。也就是说，vi 是有状态的，命令模式才能输命令，插入模式才能输入文本。这是两个基本模式。

总之命令是基础 hjkl 、i、/、G

命令模式和插入模式是两种基础模式。当你打开 vim 编辑器时所处的就是命令模式，此模式下每按下一个按键就是在输入命令序列（街机游戏玩过吧，所谓按键序列就是搓招，使用 vim 就一直在搓招，不过你操纵的不是游戏人物，而是文本），有单个按键的命令序列（比如移动光标 j，向下移动一行），多个按键的命令序列（比如向下移动光标三行 3j）。而有些按键则会进入到其他模式，

note

又是 vi 又是 vim 的是干嘛？实际上 vi 是 Visual Editor 的缩写，是诞生于 Unix 系统早期的一个文本编辑器。而 vim 则是 vi improved 的缩写，这是 Bram Moolenaar 开发的一个加强版，也是最为广泛使用的一个版本。实际上本文的所有操作都是在 vim 下进行的，vi 只是其功能的一个子集，代表着 vim 的最基础功能。

熟练掌握 vim 首先是接受它是 “模态编辑器” 的概念，然后是熟练各种模式下的文本操纵。

基础（vi and vim fundamental）​

这部分讲 vi 的基础操作，说是 vi 实际上还是用 vim，可以把这里的 vi 看作一个代表基础功能的子集。

文本编辑是我们使用电脑的常见场景，如果你用过 Microsoft Word 就应该知道，如何编辑文本：新增、编辑、重排、删除、重写错误文本等。对于程序员来说，就是源代码和各种辅助文件。

简介​

Unix 的早期出现了行编辑器（line editor），比如 ed 和 ex，被用于使用纸作为显示设备的串行终端上，之所以叫行编辑器，是因为一次只处理一或者几行文本。

随着 CRT 终端的出现（具有可定位光标），行编辑器进化为屏幕编辑器（screen editor），比如现在的 vi 和 emacs，这种编辑器可以在整块屏幕上编辑文本，并且可以在行之间快速跳转。

然后是 GUI 的出现，这些屏幕编辑器也进化为图形界面文本编辑器（graphical text editor）。可以使用鼠标到处滚动，使用光标选择要编辑的文本。（gedit、记事本等）

Unix 系统中所有标准（standard）文本编辑器中，vi 是最值得掌握的一个，几乎所有现代 Unix 系统都提供了一致形式的 vi，可以说是文本编辑的事实标准（lingua franca）。

对于初学者来说，vi 用起来很不直观——不像其他编辑器用鼠标点击就可直接编辑，vi 几乎把键盘上的每个键都当作命令，这也即命令模式（command mode），所有键击都被当作命令，而不是文本输入。你必须进入到插入模式（insert mode）才能像普通文本编辑器一样输入文本。

但一旦习惯这一点，就会明白这是一种好的设计。你只需要少量键击就能完成复杂的文本编辑任务。

vi 和 vim 不是那种“所见即所得”的富文本编辑器，如果要格式化的文档只能借助文本标记语言（markup language）。比如 Markdown 就是流行的选择，还有 LaTeX 、HTML 都是。文本标记语言能给予你远超一般字处理器的样式控制能力。比如你每天看的网页就是文本标记语言 HTML，当然样式本身就是另一个一个复杂主题。这里，我们关注“纯文本”编辑。

“编辑”通常是指对文本的操作：插入（insert）、删除（delete）、变更（change）、移动（move）、复制（copy）。这些都是文本编辑的基本操作。

和其他编辑器启动就是编辑模式不同（插入文本只占 vi 功能的一小部分），对于 vi 默认打开的是命令模式（command），复杂的编辑操作只需经过少量键击输入的命令完成，想要直接插入文本原始？使用任意一种插入（insert）命令进入插入模式即可。

  • i （Insert）在当前光标插入
  • cw（Change word）删除当前单词，并开始插入

把字母作为命令来编辑文本，可以让你以非常快的速度编辑文件。不需要记忆各种功能键，以及按那些另手指扭曲的蹩脚组合键，使用复杂的多级菜单。使用 vi 你甚至不需要把手指移开键盘，很多命令操作都很容易通过其首字母记忆，绝大部分都遵循相同的模式并且相互关联。你是在以编程的思维编辑文本

vi 命令通常有以下特点：

  • 大小写敏感
  • 不会回显到屏幕上
  • 不需要在键入命令后输入 ENTER 确认。

的确，是有一些命令是带回显的。搜索命令 / 和 ?，ex 模式命令 :，都会在底栏显示。说到 ex，本质上 vi 底层就用的它，vi 真的不过是其视觉（visual）模式而已。

补充点历史，vi 诞生于人们使用 CRT 终端作为显示器的年代，这些终端连接到一个中央迷你计算机。当时在世界范围内有几百种不同的终端，这些终端功能都大同小异（清屏、移动光标...），但用于执行这些基本动作的命令却不相同。因此当时的 Unix 系统允许用户选择要使用的控制序列，比如退格、产生中断、挂起和恢复输出等。这些基础设施都归 stty 这个命令管。

最初的 BSD Unix 使用一个叫 terminal capabilities 的文本数据库（由 termcap 库管理）。1980s System V 引入了 terminal information 的二进制数据库（由 terminfo 库管理）。这两个库的功能是相同的。为了让 vi 知道你使用的终端类型，需要设置环境变量 TERM，通常是在 .profile 和 .login 这些终端初始化文件中。

目前 tercap 库也不再使用，GNU/Linux 和 BSD 使用 ncurses 库来管理终端信息，基本上是 terminfo 库的一个兼容超集。

尽管目前我们都使用终端模拟器，但还是需要关心 TERM 环境变量。

还有一点就是，vi 诞生于 Unix 系统还不稳定的年代，那时系统崩溃是常有的事。所以 vi 提供了从意外崩溃中恢复正编辑文件的能力。当你后面见到各种错误信息时，请记住，这是历史因素使然。

使用 vi 编辑文件时，编辑器会复制一份到缓冲区（buffer），所有的编辑操作（新增、删除、修改），都在缓冲区内发生。只有当你明确执行保存命令时（"write the buffer" 或者 "writing your file"），才会写回文件。

使用 vi 编辑一个文件（vi 一般是 vim 的符号链接）

$ vim [filename]
$ vi [filename]

vi 有一个当前模式的基本概念。有两种命令模式（command mode）和插入模式（insert mode）（还有 ex 命令模式，这个后面再说）。

命令模式的每次键击都被当作命令，插入模式的每次键击都被当作输入普通文本。输入 ESC 可以回到命令模式。

保存并退出

  • ZZ

还可以进入 ex 模式保存和退出。:w 写入文件，:q 退出，:wq 保存并退出。

假设你搞乱了文件内容，想丢弃修改怎么办

:e! + ENTER

丢弃修改并退出

:q! + ENTER

覆写已存在文件

:w!

写到新文件

:w newfile
